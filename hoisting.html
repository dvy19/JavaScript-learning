Hoisting in JavaScript

Hoisting is JavaScript's behavior of moving declarations to the top of their scope during the memory creation phase, before the code is executed.
Important: Only declarations are hoisted, not initializations (assignments).

whenever javascript runs a file or function, it goes through two phases
  • Memory Creation phase
  • Execution phase

In the Memory Creation Phase, memory is allocated to variables or functions. variable with var are initializerd with undefined and functions are initialized with the entire function body and declarations are stored completely in memory.

variable declared with let and const exists but are not initialized. they are in a "temporal dead zone" from the start of the block until the initialization is processed. accessing them before the initialization results in a ReferenceError.

In the Execution Phase, the code is executed line by line. variable assignments happen functions are called.

javascript internally creates a variable with var keyword 

                var a;
                console.log(a)
                a=10;

let is hoisted, but not initialized, hence returns the refrence error. same with const.

Temporal Dead Zone

period between, the entering the scope and reaching the declarations. during this period, accessing returns RefrenceError for let and const.
